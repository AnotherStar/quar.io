# Чат с консультантом

## Назначение

`chat-consultant` — подключаемый модуль инструкций ManualOnline, добавляющий на публичную страницу инструкции плавающую кнопку чата. В текущей реализации модуль является заглушкой для будущей интеграции с внешними чат-провайдерами, такими как Crisp, Intercom или custom-виджет.

## Ключевые возможности

- Отображение фиксированной кнопки чата на публичной странице инструкции.
- Открытие и закрытие небольшого всплывающего окна по клику.
- Настройка провайдера чата через конфигурацию модуля.
- Поддержка поля `siteId` для идентификатора сайта/проекта у внешнего провайдера.
- Ограничение доступности модуля тарифом `business`.
- Декларативное описание конфигурации через `configSchema` и `configFields`.

## Архитектура

Модуль расположен в директории:

```text
instruction-modules/chat-consultant/
├── index.ts
└── Public.vue
```

### `index.ts`

Файл экспортирует объект `definition` типа `ModuleDefinition`.

Он содержит:

- `manifest` — метаданные модуля:
  - код модуля;
  - название;
  - описание;
  - версию;
  - требуемый тариф;
  - схему конфигурации;
  - описание полей конфигурации для UI.
- `PublicComponent` — ленивый импорт публичного Vue-компонента `Public.vue`.

```ts
PublicComponent: () => import('./Public.vue')
```

Это означает, что компонент публичного отображения загружается только при необходимости, когда модуль используется на странице инструкции.

### `Public.vue`

Vue-компонент отвечает за отображение виджета на публичной странице инструкции.

Компонент:

- принимает идентификатор инструкции;
- принимает конфигурацию модуля;
- принимает идентификатор сессии просмотра;
- вычисляет выбранного провайдера;
- вычисляет `siteId`;
- хранит локальное состояние открытия окна чата;
- отображает кнопку и всплывающую панель.

В текущем виде компонент не подключает внешний JavaScript SDK провайдера, а только показывает информационный блок:

```text
В продакшене сюда грузится виджет выбранного провайдера.
```

## Бизнес-логика

### Ограничение тарифа

Модуль требует тариф:

```ts
requiresPlan: 'business'
```

Это означает, что система подключения модулей должна разрешать использование `chat-consultant` только для пользователей или организаций с тарифом `business`.

В самом компоненте `Public.vue` проверка тарифа не выполняется. Ограничение задано на уровне manifest и должно обрабатываться инфраструктурой ManualOnline, отвечающей за подключение модулей.

### Выбор провайдера

Модуль поддерживает три значения провайдера:

```ts
['intercom', 'crisp', 'custom']
```

Значение по умолчанию:

```ts
'crisp'
```

Если `config.provider` отсутствует, публичный компонент также использует `crisp`.

### Отображение `siteId`

Если `siteId` указан, он выводится рядом с названием провайдера:

```vue
Провайдер: {{ provider }}<span v-if="siteId"> ({{ siteId }})</span>
```

Если `siteId` отсутствует, отображается только провайдер.

### Открытие и закрытие панели

Панель чата управляется локальным состоянием `open`.

Алгоритм:

1. Изначально `open = false`.
2. Пользователь нажимает кнопку чата.
3. Значение `open` переключается на противоположное.
4. Если `open = true`, отображается панель.
5. Если `open = false`, панель скрывается.

```vue
@click="open = !open"
```

### Интеграция с внешним чатом

Реальной загрузки SDK Crisp, Intercom или custom-виджета в текущем коде нет. Компонент содержит только текстовую заглушку:

```text
В продакшене сюда грузится виджет выбранного провайдера.
```

## З��висимости

### Внутренние зависимости ManualOnline

- `~~/modules-sdk/types` — тип `ModuleDefinition`, описывающий контракт подключаемого модуля.
- Инфраструктура подключаемых модулей инструкций — ожидается, что она:
  - читает `manifest`;
  - проверяет `requiresPlan`;
  - отображает поля настройки из `configFields`;
  - валидирует или интерпретирует `configSchema`;
  - монтирует `PublicComponent` на публичной странице инструкции.

### Vue / Nuxt

Компонент использует Vue Composition API:

- `defineProps`;
- `computed`;
- `ref`.

Импорты для `computed` и `ref` явно не указаны, что типично для Nuxt 3 при включённых auto-imports.

### Внешние сервисы

В manifest перечислены потенциальные провайдеры:

- Crisp;
- Intercom;
- Custom.

Однако в текущей реализации модуль не обращается к их API и не загружает их клиентские SDK.

## Примеры использования

### Пример конфигурации для Crisp

```json
{
  "provider": "crisp",
  "siteId": "CRISP_SITE_ID"
}
```

На публичной странице при открытии панели будет показано:

```text
Провайдер: crisp (CRISP_SITE_ID)
```

### Пример конфигурации для Intercom

```json
{
  "provider": "intercom",
  "siteId": "INTERCOM_APP_ID"
}
```

На публичной странице модуль отобразит кнопку чата, а во всплывающем окне укажет выбранного провайдера:

```text
Провайдер: intercom (INTERCOM_APP_ID)
```

## Замечания

- Модуль является заглушкой: фактическая интеграция с Crisp, Intercom или custom-провайдером не реализована.
- Props `instructionId` и `viewerSessionId` объявлены, но в текущем компоненте не используются.
- Проверка тарифа `business` не выполняется в `Public.vue`; она должна обеспечиваться инфраструктурой подключаемых модулей.
- Поле `siteId` не является обязательным на уровне `configSchema`, хотя для реальной интеграции с провайдерами оно, вероятно, потребуется.
- Компонент не выполняет валидацию `provider` на клиенте и полагается на конфигурацию, переданную платформой.
- В коде отсутствует обработка ошибок загрузки внешнего виджета, так как сама загрузка пока не реализована.
- В модуле нет серверных API-роутов, composables, сервисных функций или дополнительных схем валидации.
