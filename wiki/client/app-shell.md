# app-shell

## Назначение

`app-shell` — корневая оболочка Nuxt-приложения ManualOnline. Модуль отвечает за базовую композицию клиентского интерфейса: подключает текущий layout и отображает активную страницу маршрута через стандартные компоненты Nuxt 3.

## Ключевые возможности

- Инициализация корневого Vue/Nuxt-приложения через `app.vue`.
- Подключение системы layout-ов Nuxt с помощью `<NuxtLayout>`.
- Рендеринг текущей страницы маршрута через `<NuxtPage>`.
- Обеспечение единой точки входа для всех публичных страниц, дашборда продавца, редактора и других frontend-разделов проекта.

## Архитектура

Модуль представлен одним файлом:

```text
app.vue
```

### `app.vue`

```vue
<template>
  <NuxtLayout>
    <NuxtPage />
  </NuxtLayout>
</template>
```

`app.vue` является корневым компонентом Nuxt 3-приложения. Он не содержит собственной бизнес-логики, состояния, скриптов или стилей. Его задача — собрать стандартную иерархию Nuxt:

```text
app.vue
└── NuxtLayout
    └── NuxtPage
```

### Взаимодействие компонентов

#### `NuxtLayout`

`<NuxtLayout>` — встроенный компонент Nuxt, который выбирает и рендерит layout для текущей страницы.

Layout может быть:

- layout по умолчанию;
- layout, указанный на странице через `definePageMeta`;
- специальный layout для отдельных зон приложения, например публичной инструкции, кабинета продавца или редактора, если такие layout-ы определены в проекте.

В самом модуле `app-shell` конкретные layout-файлы не представлены.

#### `NuxtPage`

`<NuxtPage>` — встроенный компонент Nuxt, который отображает Vue-компонент текущего маршрута из файловой маршрутизации Nuxt.

Он отвечает за рендеринг страницы, соответствующей текущему URL. В ManualOnline через него могут отображаться страницы инструкций, страницы дашборда, редактора и другие разделы, если они описаны в директории страниц проекта.

В данном модуле бизнес-логика отсутствует.

`app-shell` не выполняет:

- проверку прав доступа;

## Примеры использования

### 1. Базовая структура корневого приложения

`app.vue` задаёт минимальную оболочку, через которую проходят все страницы приложения:

```vue
<template>
  <NuxtLayout>
    <NuxtPage />
  </NuxtLayout>
</template>
```

Это означает, что при переходе на любой маршрут Nuxt сначала определит подходящий layout, а затем отрендерит внутри него соответствующую страницу.

### 2. Использование layout-а на странице

Хотя в модуле `app-shell` это не реализовано напрямую, `<NuxtLayout>` позволяет страницам выбирать layout через метаданные Nuxt:

```vue
<script setup lang="ts">
definePageMeta({
  layout: 'dashboard'
})
</script>
```

В таком случае страница будет отрендерена внутри layout-а `dashboard`, если он существует в проекте.
