# page-short-url

## Назначение

Модуль `page-short-url` реализует fallback-страницу коротких публичных ссылок вида `/s/<shortId>`. Он серверно разрешает короткий идентификатор инструкции и выполняет HTTP-редирект `302` на канонический публичный URL инструкции в формате `/:tenantSlug/:instructionSlug`.

## Ключевые возможности

- Обработка публичного маршрута `/s/[shortId]` в Nuxt 3.
- Серверная загрузка данных по короткому идентификатору без клиентского “мигания” страницы.
- Редирект на канонический URL инструкции.
- Возврат ошибки `404`, если короткая ссылка не найдена или API вернул ошибку.
- Использование временного редиректа `302`.

### `pages/s/[shortId].vue`

Это динамическая страница Nuxt 3, которая обрабатывает все URL вида:

```text
/s/<shortId>
```

Файл не содержит пользовательского интерфейса в обычном смысле. Его задача — выполнить серверную логику разрешения короткой ссылки и перенаправить пользователя.

Основной поток выполнения:

1. Получить параметр маршрута `shortId` из `useRoute()`.
2. Выполнить запрос к API:

   ```ts
   /api/public/short/${shortId}
   ```

3. Проверить наличие ошибки или отсутствие данных.
4. Если данные не получены — выбросить Nuxt-ошибку `404`.
5. Если данные получены — выполнить редирект на канонический URL:

   ```text
   /<tenant.slug>/<instruction.slug>
   ```

6. Отдать HTTP-редирект с кодом `302`.

### Взаимодействие компонентов

В модуле задействованы только стандартные механизмы Nuxt:

- `useRoute()` — чтение параметра маршрута.
- `useFetch()` — серверная загрузка данных.
- `createError()` — генерация ошибки Nuxt.
- `navigateTo()` — перенаправление пользователя.

Шаблон страницы содержит пустой контейнер:

```vue
<template>
  <div />
</template>
```

Он фактически не используется для отображения данных, поскольку страница предназначена для немедленного редиректа.

### Используемый API

Модуль обращается к внутреннему публичному API:

```http
GET /api/public/short/{shortId}
```

Метод явно не указан в коде, но `useFetch()` по умолчанию выполняет `GET`-запрос.

Ожидаемый контракт ответа следует из использования в коде:

```ts
{
  tenant: {
    slug: string
  },
  instruction: {
    slug: string
  }
}
```

На основе этих данных строится канонический URL:

```ts
`/${data.value.tenant.slug}/${data.value.instruction.slug}`
```

Исходный код самого API-роута `/api/public/short/[shortId]` в предоставленном модуле отсутствует, поэтому детали его реализации, валидации и источника данных не описаны.

## Бизнес-логика

### Разрешение короткой ссылки

Короткая ссылка имеет формат:

```text
/s/<shortId>
```

После открытия такой ссылки приложение не показывает отдельную страницу, а пытается найти соответствующую инструкцию через API.

### Серверная загрузка без flash-эффекта

В коде используется:

```ts
const { data, error } = await useFetch(`/api/public/short/${shortId}`)
```

Так как запрос выполняется в `script setup` с `await`, данные запрашиваются до завершения рендера страницы. Это позволяет избежать ситуации, когда пользователь сначала видит пустую или промежуточную страницу, а затем происходит редирект.

### Обработка ошибки

Если API возвращает ошибку или данные отсутствуют:

```ts
if (error.value || !data.value) {
  throw createError({ statusCode: 404, fatal: true })
}
```

Пользователь получает страницу ошибки `404`.

Флаг `fatal: true` указывает Nuxt обработать ошибку как критическую для текущего рендера.

### Редирект на канонический URL

При успешном разрешении короткой ссылки выполняется переход:

```ts
await navigateTo(`/${data.value.tenant.slug}/${data.value.instruction.slug}`, {
  redirectCode: 302
})
```

Используется код `302`, то есть временный редирект. Это важно для случаев, когда соответствие короткой ссылки может измениться или когда не требуется закреплять permanent redirect на уровне поисковых систем и кешей.

## Зависимости

### Внутренние зависимости ManualOnline

Модуль зависит от публичного API разрешения коротких ссылок:

```text
/api/public/short/{shortId}
```

Также он косвенно зависит от модели публичной инструкции, так как ожидает получить:

- `tenant.slug`
- `instruction.slug`

Эти значения используются для построения публичного канонического URL инструкции.

### Nuxt / Vue зависимости

- `useRoute`
- `useFetch`
- `createError`
- `navigateTo`

В данном файле прямых обращений к внешним сервисам нет.

Возможные зависимости API-роута от базы данных, Prisma, PostgreSQL или других сервисов в предоставленном коде не видны.

## Примеры использования

### Сценарий 1. Переход по короткой ссылке

Пользователь открывает URL:

```text
https://manualonline.example.com/s/abc123
```

Страница `pages/s/[shortId].vue` получает:

```ts
shortId = 'abc123'
```

Затем выполняет запрос:

```http
GET /api/public/short/abc123
```

Если API возвращает:

```json
{
  "tenant": {
    "slug": "bosch"
  },
  "instruction": {
    "slug": "coffee-machine-setup"
  }
}
```

пользователь будет перенаправлен на:

```text
/bosch/coffee-machine-setup
```

с HTTP-кодом:

```text
302 Found
```

### Сценарий 2. Короткая ссылка не найдена

Пользователь открывает:

```text
/s/unknown-id
```

Если API возвращает ошибку или пустой результат, модуль выбрасывает ошибку:

```ts
throw createError({ statusCode: 404, fatal: true })
```

В результате пользователь увидит страницу `404`.

## Замечания

- В модуле присутствует только Nuxt-страница. Реализация API `/api/public/short/{shortId}` не предоставлена, поэтому невозможно описать точные правила поиска короткой ссылки, схему данных и обработку статусов публикации.
- Тип ответа `useFetch()` явно не задан. Сейчас структура `data.value.tenant.slug` и `data.value.instruction.slug` выводится не из локального TypeScript-контракта, а из фактического использования.
- Используется временный редирект `302`. Если короткие ссылки должны быть постоянными с точки зрения SEO, может потребоваться отдельное решение о применении `301`, но в текущем коде явно выбран `302`.
- Шаблон содержит только `<div />`, поскольку пользовательский интерфейс не требуется. Основная логика выполняется до отображения страницы.
