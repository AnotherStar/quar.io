# page-auth

## Назначение

Модуль `page-auth` содержит Nuxt-страницы аутентификации, расположенные в `pages/auth`. Он отвечает за пользовательские сценарии входа и регистрации в ManualOnline, включая создание аккаунта продавца и переход в дашборд после успешной авторизации.

## Ключевые возможности

- Страница входа пользователя по email и паролю: `/auth/login`.
- Страница регистрации пользователя и компании: `/auth/register`.
- Использование отдельного `blank`-layout для auth-страниц.
- Защита страниц middleware `guest`, чтобы авторизованные пользователи не проходили повторную авторизацию.
- Отправка данных на серверные API:
  - `POST /api/auth/login`
  - `POST /api/auth/register`
- Обновление клиентского состояния авторизации через `useAuthState().refresh()`.
- Перенаправление пользователя в `/dashboard` после успешного входа или регистрации.
- Отображение ошибок API через `UiAlert`.
- Индикация загрузки на кнопках отправки формы.
- Автогенерация slug компании на странице регистрации на основе названия компании.

## Архитектура

Модуль состоит из двух Nuxt page-компонентов:

```text
pages/
└── auth/
    ├── login.vue
    └── register.vue
```

### `pages/auth/login.vue`

Страница входа пользователя.

Основные элементы:

- `definePageMeta({ layout: 'blank', middleware: 'guest' })`
  - использует layout `blank`;
  - подключает middleware `guest`.
- Локальное состояние формы:

```ts
const form = reactive({ email: '', password: '' })
```

- Состояние ошибки:

```ts
const error = ref<string | null>(null)
```

- Состояние загрузки:

```ts
const loading = ref(false)
```

- Получение метода обновления auth-состояния:

```ts
const { refresh } = useAuthState()
```

- Отправка формы выполняется функцией `submit()`:
  1. сбрасывает предыдущую ошибку;
  2. включает состояние загрузки;
  3. отправляет `POST`-запрос на `/api/auth/login`;
  4. вызывает `refresh()` для обновлен��я клиентского состояния авторизации;
  5. перенаправляет пользователя на `/dashboard`;
  6. при ошибке показывает сообщение из ответа API или fallback-текст.

Шаблон использует UI-компоненты:

- `NuxtLink`
- `UiInput`
- `UiAlert`
- `UiButton`

### `pages/auth/register.vue`

Страница регистрации пользователя и компании.

Основные элементы:

- `definePageMeta({ layout: 'blank', middleware: 'guest' })`
  - страница также использует `blank`-layout;
  - доступ ограничен middleware `guest`.

- Локальное состояние формы:

```ts
const form = reactive({
  email: '',
  password: '',
  name: '',
  tenantName: '',
  tenantSlug: ''
})
```

- Поля формы:
  - `email` — email пользователя;
  - `password` — пароль;
  - `name` — имя пользователя;
  - `tenantName` — название компании;
  - `tenantSlug` — slug компании.

- Автогенерация `tenantSlug`:

```ts
watch(() => form.tenantName, (n) => {
  if (!form.tenantSlug) {
    form.tenantSlug = n
      .toLowerCase()
      .replace(/[^a-z0-9]+/g, '-')
      .replace(/^-+|-+$/g, '')
      .slice(0, 40)
  }
})
```

Slug генерируется только если поле `tenantSlug` ещё пустое. Это позволяет пользователю вручную переопределить slug, после чего автоматическая генерация больше не будет перезаписывать введённое значение.

- Отправка формы выполняется через `POST /api/auth/register`.
- После успешной регистрации:
  1. обновляется auth-состояние через `refresh()`;
  2. пользователь перенаправляется в `/dashboard`.

## Бизнес-логика

### Вход

Сценарий входа:

1. Пользователь вводит email и пароль.
2. Форма отправляется без перезагрузки страницы через `@submit.prevent`.
3. Клиент отправляет запрос:

```ts
await $fetch('/api/auth/login', {
  method: 'POST',
  body: form
})
```

4. После успешного ответа вызывается:

```ts
await refresh()
```

5. Затем пользователь перенаправляется:

```ts
await navigateTo('/dashboard')
```

6. Если запрос завершился ошибкой, отображается сообщение:

```ts
e?.data?.statusMessage ?? e?.statusMessage ?? 'Не удалось войти'
```

### Регистрация

Сценарий регистрации:

1. Пользователь вводит email, пароль, имя, название компании и slug компании.
2. Пароль сопровождается подсказкой:

```text
Минимум 8 символов
```

Проверка минимальной длины пароля на клиенте в коде не реализована. Это только UI-подсказка.

3. При вводе названия компании автоматически формируется `tenantSlug`, если он ещё не заполнен вручную.
4. Slug формируется по правилам:
   - перевод в нижний регистр;
   - замена всех последовательностей символов, кроме `a-z` и `0-9`, на дефис;
   - удаление дефисов в начале и конце;
   - ограничение длины до 40 символов.

Алгоритм:

```ts
n.toLowerCase()
  .replace(/[^a-z0-9]+/g, '-')
  .replace(/^-+|-+$/g, '')
  .slice(0, 40)
```

5. Форма отправляется:

```ts
await $fetch('/api/auth/register', {
  method: 'POST',
  body: form
})
```

6. После успешной регистрации вызывается `refresh()`.
7. Пользователь перенаправляется в `/dashboard`.
8. При ошибке показывается сообщение:

```ts
e?.data?.statusMessage ?? e?.statusMessage ?? 'Не удалось зарегистрироваться'
```

## Примеры использования

### Вход пользователя

Пользователь открывает страницу:

```text
/auth/login
```

Вводит email и пароль. После отправки формы клиент выполняет:

```ts
await $fetch('/api/auth/login', {
  method: 'POST',
  body: {
    email: form.email,
    password: form.password
  }
})

await refresh()
await navigateTo('/dashboard')
```

Если API возвращает ошибку, она отображается в форме через `UiAlert`.

### Регистрация компании

Пользователь открывает:

```text
/auth/register
```

Заполняет название компании, например:

```text
My Test Company
```

Если поле slug ещё пустое, клиент автоматически сформирует:

```text
my-test-company
```

После отправки формы данные уходят на сервер:

```ts
await $fetch('/api/auth/register', {
  method: 'POST',
  body: {
    email: form.email,
    password: form.password,
    name: form.name,
    tenantName: form.tenantName,
    tenantSlug: form.tenantSlug
  }
})
```

После успешного ответа пользователь попадает в `/dashboard`.

## Замечания

- В предоставленном коде отсутствуют реализации:
  - `/api/auth/login`;
  - `/api/auth/register`;
  - `useAuthState`;
  - middleware `guest`;
  - layout `blank`;
  - UI-компонентов `UiInput`, `UiButton`, `UiAlert`.

- Клиентская валидация минимальной длины пароля не реализована. На странице регистрации есть только подсказка `Минимум 8 символов`.

- Автогенерация `tenantSlug` поддерживает только латинские буквы `a-z`, цифры и дефисы. Кириллица и другие символы будут заменены на дефис и могут привести к пустому slug, если название компании не содержит латиницу или цифры.

- `tenantSlug` автоматически обновляется только пока поле пустое. Если пользователь начал редактировать slug вручную, дальнейшие изменения `tenantName` не будут менять slug.

- Обработка ошибок зависит от структуры ошибки `$fetch`:

```ts
e?.data?.statusMessage ?? e?.statusMessage
```

Если сервер возвращает ошибку в другом формате, пользователь увидит fallback-сообщение.

- Объект `form` передаётся в `$fetch` напрямую. Это штатно работает с reactive-объектом Vue, но при усложнении формы может быть предпочтительнее отправлять plain object с явно выбранными полями.
