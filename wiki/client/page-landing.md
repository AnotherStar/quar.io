# page-landing

## Назначение

`page-landing` — это главная публичная страница quar.io, реализованная как Nuxt page `pages/index.vue`. Модуль отвечает за первое представление продукта: объясняет ценность сервиса, показывает ключевые возможности и направляет пользователя к регистрации или просмотру тарифов.

## Ключевые возможности

- Отображение hero-блока с позиционированием продукта quar.io.
- SEO-заголовок страницы через `useHead`.
- CTA-кнопки для перехода к регистрации и тарифам.
- Декоративный mockup карточки публичной инструкции.
- Секция с описанием ключевых возможностей платформы.
- Финальный CTA-блок для быстрого создания аккаунта.
- Использование внутренних UI-компонентов `UiButton` и `UiCard`.
- Адаптивная сетка карточек возможностей через utility-классы.

## Архитектура

### Файловая структура

Модуль представлен одним файлом:

```text
pages/
└── index.vue
```

### `pages/index.vue`

Это Nuxt page-компонент, который автоматически регистрируется Nuxt как маршрут главной страницы:

```text
GET /
```

Компонент состоит из двух частей:

```vue
<script setup lang="ts">
useHead({ title: 'quar.io — инструкции, которые работают' })
</script>
```

и шаблона:

```vue
<template>
  ...
</template>
```

### Основные секции страницы

#### 1. Hero-секция

```vue
<section class="bg-navy text-white">
```

Содержит:

- короткий caption: `SaaS для производителей и продавцов`;
- основной заголовок;
- описание продукта;
- две CTA-кнопки:
  - `/auth/register` — регистрация;
  - `/pricing` — тарифы;
- mockup интерфейса инструкции.

Hero-блок оформлен тёмным фоном `bg-navy` и белым текстом.

#### 2. Mock workspace card

Внутри hero-секции расположен статический визуальный макет страницы инструкции:

```vue
<div class="mx-auto mt-16 max-w-4xl rounded-lg bg-canvas text-ink shadow-mockup">
```

Он имитирует карточку браузерного окна с адресом:

```text
manual.online/atvel/f-16
```

и примером инструкции:

```text
F-16 · Сборка модели 1:48
```

Макет не содержит интерактивной логики и служит только демонстрацией продукта.

#### 3. Секция возможностей

```vue
<section id="features" class="container-page py-section-lg">
```

Содержит заголовок:

```text
Всё, что нужно для современной инструкции
```

и шесть карточек `UiCard`:

1. Notion-like редактор
2. История версий
3. Аналитика и отзывы
4. Workflow одобрения
5. Подключаемые модули
6. Не платите — ссылки живут

Карточки выводятся статически в адаптивной сетке:

```vue
<div class="mt-section grid grid-cols-1 gap-md md:grid-cols-3">
```

На мобильных устройствах карточки идут в одну колонку, начиная с `md` — в три колонки.

#### 4. Финальная CTA-секция

```vue
<section class="bg-surface">
```

Содержит:

- заголовок `Начните за 2 минуты`;
- пояснение `Бесплатно — до трёх инструкций`;
- кнопку перехода к регистрации `/auth/register`.

## API / Интерфейс

### Nuxt route

Файл `pages/index.vue` создаёт клиентский/SSR-маршрут Nuxt:

| Route | Метод | Назначение |
|---|---:|---|
| `/` | `GET` | Главная landing-страница quar.io |

Отдельных Nitro API route-файлов в данном модуле нет.

### Props

`pages/index.vue` не принимает props.

### Events

Компонент не эмитит пользовательские события.

### Slots

Компонент не объявляет slots.

### Используемые компоненты

#### `UiButton`

Используется для навигационных CTA.

Примеры из кода:

```vue
<UiButton to="/auth/register" size="md">Начать бесплатно</UiButton>
```

```vue
<UiButton to="/pricing" variant="secondary-on-dark" size="md">Тарифы</UiButton>
```

```vue
<UiButton to="/auth/register" class="mt-6">Создать аккаунт</UiButton>
```

Из текущего файла видно использование следующих параметров:

| Prop / атрибут | Пример | Назначение |
|---|---|---|
| `to` | `"/auth/register"` | Целевой маршрут для перехода |
| `size` | `"md"` | Размер кнопки |
| `variant` | `"secondary-on-dark"` | Вариант оформления |
| `class` | `"mt-6"` | Дополнительные CSS-классы |

Точный контракт `UiButton` в данном фрагменте не определён, так как исходный код компонента не предоставлен.

#### `UiCard`

Используется для карточек возможностей.

Пример:

```vue
<UiCard tint="lavender" :bordered="false">
  <h3 class="text-h4">Notion-like редактор</h3>
  <p class="mt-2 text-body text-charcoal/80">
    Заголовки, списки, картинки, видео, спец-блоки безопасности — без программистов.
  </p>
</UiCard>
```

Из текущего файла видно использование следующих параметров:

| Prop | Пример | Назначение |
|---|---|---|
| `tint` | `"lavender"` | Цветовой вариант карточки |
| `bordered` | `false` | Отключение рамки |

Контент карточки передаётся через default slot.

Точный контракт `UiCard` в данном фрагменте не определён, так как исходный код компонента не предоставлен.

### Composables

#### `useHead`

Используется для установки заголовка HTML-документа:

```ts
useHead({ title: 'quar.io — инструкции, которые работают' })
```

Это стандартный composable Nuxt 3.

## Бизнес-логика

В модуле отсутствует прикладная бизнес-логика в виде вычислений, API-запросов, проверок прав, работы с тарифами или публикацией инструкций.

Фактически реализованы только статические маркетинговые сценарии:

1. **Переход к регистрации**

   Пользователь может перейти на страницу регистрации из hero-блока или финального CTA:

   ```vue
   <UiButton to="/auth/register">...</UiButton>
   ```

2. **Переход к тарифам**

   В hero-блоке есть ссылка на страницу тарифов:

   ```vue
   <UiButton to="/pricing" variant="secondary-on-dark" size="md">Тарифы</UiButton>
   ```

3. **Декларация бесплатного лимита**

   В финальной секции указано:

   ```text
   Бесплатно — до трёх инструкций.
   ```

   В данном файле это только текстовое утверждение. Проверки тарифа, лимитов или регистрации здесь не реализованы.

4. **Описание возможностей платформы**

   Секция `features` перечисляет возможности quar.io, но не подключает реальные данные, API или конфигурацию модулей.

## Зависимости

### Внутренние зависимости quar.io

Модуль зависит от следующих внутренних UI-элементов и дизайн-системы проекта:

- `UiButton` — кнопки с поддержкой навигации.
- `UiCard` — карточки для секции возможностей.
- CSS utility-классы проекта:
  - `container-page`;
  - `py-hero`;
  - `py-section`;
  - `py-section-lg`;
  - `text-hero-display`;
  - `text-subtitle`;
  - `text-caption-bold`;
  - `text-h2`;
  - `text-h4`;
  - `shadow-mockup`;
  - цветовые токены вроде `bg-navy`, `text-ink`, `bg-canvas`, `bg-surface`, `text-charcoal`.

По коду видно, что страница опирается на настроенную дизайн-систему/Tailwind utility layer проекта.

### Внешние зависимости

- Nuxt 3:
  - файловый роутинг через `pages/index.vue`;
  - composable `useHead`.
- Vue 3:
  - `<script setup lang="ts">`;
  - single-file component syntax.
- CSS/Tailwind-подобные utility-классы.

### Не используются

В данном модуле не используются:

- Nitro API routes;
- Prisma;
- PostgreSQL;
- S3/MinIO;
- TipTap;
- zod-схемы;
- серверные сервисы;
- client-side state management;
- асинхронная загрузка данных.

## Примеры использования

### Переход пользователя к регистрации

Пользователь открывает главную страницу:

```text
/
```

В hero-блоке нажимает:

```text
Начать бесплатно
```

Кнопка ведёт на маршрут:

```text
/auth/register
```

Фрагмент реализации:

```vue
<UiButton to="/auth/register" size="md">
  Начать бесплатно
</UiButton>
```

### Добавление новой карточки возможности

Если требуется добавить ещё одну статическую возможность в секцию `features`, можно добавить новый `UiCard` внутрь существующей grid-сетки:

```vue
<UiCard tint="mint" :bordered="false">
  <h3 class="text-h4">Новая возможность</h3>
  <p class="mt-2 text-body text-charcoal/80">
    Краткое описание возможности платформы.
  </p>
</UiCard>
```

При этом нужно убедиться, что значение `tint` поддерживается компонентом `UiCard`.

## Замечания

- Страница полностью статическая: данные, тексты и карточки возможностей захардкожены в шаблоне.
- Нет локализации/i18n: весь текст находится непосредственно в `index.vue`.
- Нет обработки состояний загрузки, ошибок или feature flags.
- Mockup инструкции не связан с реальными инструкциями из базы данных.
- Заявление `Бесплатно — до трёх инструкций` не подкреплено проверкой тарифа в этом модуле; фактическая тарифная логика должна находиться в других частях системы.
- Контракты `UiButton` и `UiCard` в исходном коде страницы используются, но их реализация не предоставлена, поэтому точные props/events этих компонентов не могут быть полностью описаны по данному фрагменту.
- В коде не видно TODO-комментариев или явных признаков технического долга, кроме статичности контента.
