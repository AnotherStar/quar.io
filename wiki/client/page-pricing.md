# page-pricing

## Назначение

`page-pricing` — это Nuxt-страница `pages/pricing.vue`, отображающая публичную страницу тарифов quar.io по маршруту `/pricing`. Модуль показывает доступные тарифные планы, их стоимость, основные ограничения и преимущества, а также направляет пользователя к регистрации.

## Ключевые возможности

- Отображение трёх тарифных планов: `Free`, `Plus`, `Business`.
- Визуальное выделение рекомендуемого тарифа `Plus`.
- Вывод стоимости, периода оплаты и списка возможностей для каждого тарифа.
- CTA-кнопка для перехода на регистрацию.
- Установка SEO/браузерного заголовка страницы через `useHead`.
- Адаптивная сетка тарифов: одна колонка на мобильных экранах и три колонки на `md`.

### Структура модуля

Модуль реализован как single-file component Vue 3 с использованием `<script setup lang="ts">`.

```vue
<script setup lang="ts">
useHead({ title: 'Тарифы — quar.io' })

const tiers = [...]
</script>
```

Внутри страницы определён локальный массив `tiers`, который является единственным источником данных для карточек тарифов.

### Маршрутизация

Так как файл расположен в директории `pages`, Nuxt автоматически создаёт маршрут:

```text
/pricing
```

Страница является публичной и не содержит в коде проверок авторизации, загрузки данных с сервера или middleware.

### Компонентная структура

В шаблоне используются:

- контейнер страницы;
- заголовок и подзаголовок;
- grid-контейнер карточек тарифов;
- карточки тарифов, отрисованные через `v-for`;
- внутренние UI-компоненты:
  - `UiBadge`;
  - `UiButton`.

Упрощённая структура:

```text
pages/pricing.vue
└── div.container-page
    ├── h1
    ├── p
    └── div.grid
        └── div[tier card] x tiers.length
            ├── h3
            ├── UiBadge, если тариф featured
            ├── price
            ├── subtitle
            ├── UiButton
            └── ul features
```

### Данные тарифов

Массив `tiers` содержит объекты следующей фактической формы:

```ts
type PricingTier = {
  code: 'free' | 'plus' | 'business'
  name: string
  price: string
  sub: string
  featured?: boolean
  features: string[]
}
```

В коде тип явно не объявлен, но такая структура следует из используемых полей.

### Используемые UI-компоненты

#### `UiBadge`

Используется для отображения метки популярного тарифа:

```vue
<UiBadge v-if="t.featured" variant="popular">
  Популярно
</UiBadge>
```

Из текущего кода видно использование prop:

| Prop | Значение | Назначение |
|---|---|---|
| `variant` | `"popular"` | Визуальный стиль бейджа |

Точный контракт `UiBadge` в предоставленном коде не раскрыт.

#### `UiButton`

Используется как CTA-кнопка перехода на регистрацию:

```vue
<UiButton
  :to="t.code === 'free' ? '/auth/register' : '/auth/register'"
  class="mt-md"
  block
  :variant="t.featured ? 'primary' : 'secondary'"
>
  {{ t.code === 'free' ? 'Начать' : 'Выбрать' }}
</UiButton>
```

Из текущего кода видно использование props:

| Prop | Значение | Назначение |
|---|---|---|
| `to` | `'/auth/register'` | Целевой маршрут перехода |
| `block` | `true` | Отображение кнопки на всю ширину |
| `variant` | `'primary'` или `'secondary'` | Визуальный вариант кнопки |

Точный контракт `UiButton` в предоставленном коде не раскрыт.

## Бизнес-логика

Бизнес-логика страницы полностью статическая и находится в массиве `tiers`.

### Тариф Free

```ts
{
  code: 'free',
  name: 'Free',
  price: '0 ₽',
  sub: 'навсегда'
```

```text
/auth/register
```

- кнопка использует вариант `primary`.

CTA-текст:

```text

```

Переход:

```text
/auth/register
```

Переход:

```text

```

### Правила отображения

- Карточки тарифов создаются через `v-for="t in tiers"`.
- Ключ карточки — `t.code`.
- Если у тарифа установлен `featured: true`, карточка получает выделенный стиль.
- Бейдж `Популярно` показывается только для featured-тарифа.
- Вариант кнопки:
  - `primary` для featured-тарифа;
  - `secondary` для остальных.
- Текст кнопки:
  - `Начать` для тарифа `free`;
  - `Выбрать` для остальных тарифов.
- Все тарифы ведут на один и тот же маршрут регистрации `/auth/register`.

## Примеры использования

Публичная страница доступна по маршруту:

Чтобы добавить возможность в существующий тариф, нужно расширить массив `features` соответствующего объекта:

```ts
{
  code: 'plus',
  name: 'Plus',
  price: '1 900 ₽',
  sub: 'в месяц',
  featured: true,
  features: [
    'До 50 инструкций',

    'Модуль гарантии',
    'Workflow одобрения',
    'Аналитика 1 год',
    'До 3 пользователей',
    'Новая возможность'

```

## Замечания

- Данные тарифов захардкожены в `pages/pricing.vue`; отдельного API, CMS, конфигурационного файла или интеграции с биллингом нет.
- Все CTA-кнопки ведут на один и тот же маршрут `/auth/register`.
- Выражение для `to` в `UiButton` избыточно:

  ```vue
  :to="t.code === 'free' ? '/auth/register' : '/auth/register'"
  ```

  Обе ветки возвращают одинаковое значение. Его можно заменить на:

  ```vue
  to="/auth/register"
  ```

- В коде нет передачи выбранного тарифа на страницу регистрации. После клика невозможно определить, какой тариф выбрал пользователь, если это не реализовано в другом месте проекта.
- Упоминание `API + webhooks (скоро)` является только текстом в списке возможностей; фактическая реализация API/webhooks в этом модуле отсутствует.
- Тип `PricingTier` явно не объявлен. Для большей поддерживаемости можно вынести тип и данные тарифов в отдельный файл конфигурации.
- В предоставленном фрагменте нет локализации/i18n: все тексты тарифов находятся непосредственно в компоненте.
