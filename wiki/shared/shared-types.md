# shared-types

## Назначение

Модуль `shared-types` содержит общие TypeScript-типы и небольшие утилиты, используемые одновременно на клиенте и сервере quar.io. В текущей реализации он описывает минимальный контракт JSON-документа TipTap/ProseMirror для инструкций и предоставляет функции для извлечения идентификаторов блоков и plain text из документа.

## Ключевые возможности

- Описание общего формата TipTap-документа через интерфейсы `TiptapDoc` и `TiptapNode`.
- Предоставление константы `EMPTY_DOC` для безопасной инициализации пустого документа инструкции.
- Сбор `id` атрибутов из узлов TipTap-документа через `collectBlockIds`.
- Извлечение plain text из документа через `extractText`.
- Использование одного контракта данных на клиенте и сервере без дублирования типов.

## Архитектура

Модуль представлен одним файлом:

```text
shared/types/instruction.ts
```

### `shared/types/instruction.ts`

Файл содержит:

- TypeScript-интерфейсы для описания структуры TipTap-документа:
  - `TiptapNode`
  - `TiptapDoc`
- Константу:
  - `EMPTY_DOC`
- Сервисные функции для обхода дерева документа:
  - `collectBlockIds`
  - `extractText`

Модуль не содержит Vue-компонентов, composables, Nitro API-роутов или zod-схем. Это общий слой контрактов и утилит, предназначенный для переиспользования в разных частях приложения: публичных страницах инструкций, редакторе, серверной обработке и аналитике.

Структура документа намеренно описана «мягко»: типы отражают базовую форму ProseMirror/TipTap JSON, но не пытаются строго валидировать все возможные расширения TipTap. Это важно, потому что набор расширений редактора может меняться, а shared-контракт должен оставаться достаточно стабильным.

## API / Интерфейс

### Интерфейс `TiptapNode`

```ts
export interface TiptapNode {
  type: string
  attrs?: Record<string, unknown>
  content?: TiptapNode[]
  text?: string
  marks?: Array<{ type: string; attrs?: Record<string, unknown> }>
}
```

Описывает отдельный узел TipTap/ProseMirror-документа.

Поля:

| Поле | Тип | Обязательное | Описание |
|---|---|---:|---|
| `type` | `string` | Да | Тип узла, например `paragraph`, `text`, `heading` или другой тип, определённый расширениями TipTap. |
| `attrs` | `Record<string, unknown>` | Нет | Атрибуты узла. В текущем модуле используется, в частности, `attrs.id` для идентификации блоков. |
| `content` | `TiptapNode[]` | Нет | Дочерние узлы. Используется для рекурсивного обхода документа. |
| `text` | `string` | Нет | Текстовое содержимое текстового узла. |
| `marks` | `Array<{ type: string; attrs?: Record<string, unknown> }>` | Нет | Mark-обёртки TipTap/ProseMirror, например форматирование текста. |

### Интерфейс `TiptapDoc`

```ts
export interface TiptapDoc {
  type: 'doc'
  content?: TiptapNode[]
}
```

Описывает корневой документ TipTap.

Поля:

| Поле | Тип | Обязательное | Описание |
|---|---|---:|---|
| `type` | `'doc'` | Да | Корневой тип документа. |
| `content` | `TiptapNode[]` | Нет | Список верхнеуровневых узлов документа. |

### Константа `EMPTY_DOC`

```ts
export const EMPTY_DOC: TiptapDoc = { type: 'doc', content: [] }
```

Пустой документ TipTap, используемый как безопасное начальное значение.

Назначение:

- инициализация редактора без контента;
- fallback-значение при создании новой инструкции;
- избежание `undefined` при работе с документом.

### Функция `collectBlockIds`

```ts
export function collectBlockIds(doc: TiptapDoc): string[]
```

Рекурсивно обходит TipTap-документ и собирает значения `attrs.id` у всех узлов, где этот атрибут является строкой.

Использование по назначению из комментария в коде:

- аналитика событий просмотра блоков, например `BLOCK_VIEW`;
- таргетинг виджета обратной связи на конкретный блок инструкции.

Алгоритм:

1. Создаётся пустой массив `ids`.
2. Для каждого верхнеуровневого узла документа запускается рекурсивный обход.
3. Если у узла есть `attrs.id` и значение имеет тип `string`, оно добавляется в массив.
4. Если у узла есть `content`, функция обходит дочерние узлы.
5. Возвращается массив найденных идентификаторов.

Особенности:

- функция не проверяет уникальность `id`;
- порядок идентификаторов соответствует порядку обхода дерева документа;
- нестроковые значения `attrs.id` игнорируются.

### Функция `extractText`

```ts
export function extractText(doc: TiptapDoc): string
```

Извлекает весь plain text из TipTap-документа.

Использование по назначению из комментария в коде:

- построение поискового индекса внутри инструкции.

Алгоритм:

1. Создаётся массив строк `out`.
2. Документ обходится рекурсивно.
3. Если у узла есть поле `text`, значение добавляется в `out`.
4. Если у узла есть `content`, обход продолжается по дочерним узлам.
5. Все найденные текстовые фрагменты объединяются через пробел.

Особенности:

- форматирование, marks и attrs не учитываются;
- структура документа не сохраняется;
- между всеми текстовыми фрагментами вставляется один пробел.

## Бизнес-логика

Модуль содержит ограниченную бизнес-логику, связанную с обработкой контента инструкции.

### Идентификаторы блоков

Функция `collectBlockIds` рассматривает любой узел с атрибутом `attrs.id` типа `string` как блок, пригодный для аналитики или таргетинга feedback-виджета.

Правила:

- учитываются только строковые `id`;
- узел может находиться на любом уровне вложенности;
- проверка схемы узла не выполняется;
- дубликаты не удаляются.

Это означает, что ответственность за генерацию корректных и уникальных `id` лежит на коде редактора или серверной логике, которая формирует документ.

### Извлечение текста для поиска

Функция `extractText` формирует упрощённое текстовое представление инструкции.

Правила:

- учитываются только поля `text`;
- текст извлекается из всех уровней вложенности;
- результат объединяется пробелами;
- marks, attrs, типы узлов и семантика блоков не влияют на итоговую строку.

Такой подход подходит для базового индексирования, но не сохраняет заголовки, секции, порядок блоков как отдельную структуру или вес элементов.

### Гибкая структура TipTap

Типы `TiptapNode` и `TiptapDoc` не навязывают строгую схему конкретных расширений TipTap. Это сделано намеренно, поскольку набор расширений редактора может развиваться.

Следствие:

- shared-контракт остаётся совместимым с разными типами узлов;
- структурная валидация документа должна выполняться в других слоях, если она требуется;
- модуль не гарантирует, что документ валиден с точки зрения конкретной конфигурации TipTap.

## Зависимости

### Внутренние зависимости

Явных импортов из других модулей quar.io в файле нет.

Логически модуль связан со следующими частями системы:

- редактором инструкций на TipTap;
- публичным рендерингом инструкций;
- аналитикой блоков инструкции;
- механизмом обратной связи по конкретным блокам;
- поиском внутри инструкции.

Эти связи следуют из комментариев и назначения функций, но прямые зависимости в коде отсутствуют.

### Внешние зависимости

Внешние npm-пакеты или сервисы в модуле не используются.

Модуль опирается только на стандартные возможности TypeScript/JavaScript:

- интерфейсы TypeScript;
- массивы;
- рекурсивный обход объектов;
- optional chaining.

## Примеры использования

### Инициализация пустой инструкции

```ts
import { EMPTY_DOC, type TiptapDoc } from '~/shared/types/instruction'

const doc: TiptapDoc = EMPTY_DOC
```

Такой документ можно использовать как начальное значение для новой инструкции или как fallback при отсутствии сохранённого контента.

### Сбор идентификаторов блоков для аналитики

```ts
import { collectBlockIds, type TiptapDoc } from '~/shared/types/instruction'

const doc: TiptapDoc = {
  type: 'doc',
  content: [
    {
      type: 'paragraph',
      attrs: { id: 'block-intro' },
      content: [
        {
          type: 'text',
          text: 'Перед началом работы прочитайте инструкцию.'
        }
      ]
    },
    {
      type: 'paragraph',
      attrs: { id: 'block-step-1' },
      content: [
        {
          type: 'text',
          text: 'Подключите устройство к питанию.'
        }
      ]
    }
  ]
}

const blockIds = collectBlockIds(doc)
// ['block-intro', 'block-step-1']
```

### Извлечение текста для поиска

```ts
import { extractText, type TiptapDoc } from '~/shared/types/instruction'

const doc: TiptapDoc = {
  type: 'doc',
  content: [
    {
      type: 'heading',
      content: [{ type: 'text', text: 'Установка устройства' }]
    },
    {
      type: 'paragraph',
      content: [{ type: 'text', text: 'Закрепите устройство на стене.' }]
    }
  ]
}

const text = extractText(doc)
// 'Установка устройства Закрепите устройство на стене.'
```

## Замечания

- В модуле намеренно отсутствует строгая структурная валидация TipTap-документа. Комментарий в коде указывает, что это сделано из-за эволюции TipTap-расширений.
- `collectBlockIds` не удаляет дубликаты. Если в документе несколько узлов имеют одинаковый `attrs.id`, функция вернёт повторяющиеся значения.
- `collectBlockIds` не проверяет формат идентификатора блока, только тип `string`.
- `extractText` не учитывает семантику узлов, форматирование и структуру документа. Все найденные текстовые фрагменты объединяются через пробел.
- В текущем исходном коде представлен только один файл модуля. API-роуты, Vue-компоненты, zod-схемы или SDK-клиенты в предоставленном фрагменте отсутствуют.
